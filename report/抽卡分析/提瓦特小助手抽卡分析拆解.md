# 提瓦特小助手风格抽卡分析拆解（面向实现）

更新时间：2026-04-14

## 1. 目标
你给的示例图本质上是“抽卡数据画像卡片”：
- 上半区：一句话运势结论 + 关键指标
- 中间区：阶段总结（限定/常驻 5 星统计）
- 下半区：分卡池时间线与每个 5 星出货抽数条形图

这类结果对用户的核心价值是：
- 让玩家快速知道自己是“欧还是非”
- 把“保底状态、歪没歪、平均几抽出金”可视化
- 把角色池/武器池拆开，便于下一次抽卡决策

## 2. 示例图可还原的功能模块

### 2.1 总览头图（主卡）
可见字段：
- 特征码（用于分享检索）
- 运势等级文案（如“超级至尊非酋”）
- 总抽卡数
- 平均出金（每个 5 星平均消耗抽数）
- 标签（如“常吃保底”“武器100%不歪”）
- 小保底不歪率
- 五星数
- 每 UP 角色平均抽数
- 每 UP 武器平均抽数

玩家看到的效果：
- 10 秒内看懂账号抽卡“手气画像”
- 知道自己到底是“偶尔非”还是“长期非”

### 2.2 抽卡总结区
可见字段：
- 限定五星数量
- 常驻五星数量
- 具体获得名单（角色/武器头像列表）

玩家看到的效果：
- 一眼回顾“历史产出质量”
- 判断抽卡资源是否转化成了想要的限定资产

### 2.3 卡池分栏与出货时间线
可见字段：
- 角色唤取 / 武器唤取 / 常驻角色 / 常驻武器 / 新手
- 每次 5 星对应“几抽出”的柱状条
- 当前垫池进度（已垫 N 抽）
- 歪/定标记（如红色“歪”、定轨成功标记）

玩家看到的效果：
- 明确“下一个 5 星大概还要多少抽”
- 复盘每次出金是否保底、是否偏非

## 3. 指标定义（建议统一口径）

### 3.1 基础统计
1. 总抽卡数
- 定义：所有卡池记录条数之和

2. 五星数
- 定义：`rank=5` 的记录条数

3. 平均出金
- 定义：
  - 先按卡池计算相邻 5 星间隔抽数（包含该次出金）
  - 全部间隔求平均：`avg_pity = sum(intervals) / count(5star)`

### 3.2 “歪/不歪”与保底
1. 角色池小保底不歪率
- 定义：在“非大保底”状态下触发 5 星时，命中当期 UP 角色的比例
- 公式：`win_5050_rate = win_5050_count / total_5050_trials`

2. 大保底命中率（可选）
- 定义：上一金歪后，下一金是否命中当期 UP

3. 武器定轨成功率（可选）
- 定义：在武器池定轨场景下，最终命中目标武器的比例

说明：
- 这些指标需要“当期 UP 列表 + 卡池期次”元数据，不能只靠抽卡记录文本名裸算。

### 3.3 成本指标
1. 每 UP 角色需抽
- 定义：角色活动池总抽数 / 获得的当期 UP 五星角色数量

2. 每 UP 武器需抽
- 定义：武器活动池总抽数 / 获得的当期目标 UP 五星武器数量

### 3.4 运势等级映射（文案系统）
建议将“运势等级”做成可配置阈值映射：
- 输入：`avg_pity`, `win_5050_rate`, `early_gold_rate`
- 输出：如“终极欧皇 / 小欧 / 平民 / 小非 / 大非 / 超级至尊非酋”

这样后续可根据真实样本分布调参，而不是写死。

## 4. 实现这类分析必须补齐的数据

1. 抽卡流水（你当前 CLI 已具备）
- 时间、名称、星级、池子类型、记录 ID

2. 卡池期次与 UP 元数据（新增）
- 每期开始/结束时间
- 当期 UP 五星角色/武器列表
- 池子类型（301/302/...）

3. 角色/武器图标资源（新增）
- 用于生成“头像墙”和柱状图左侧图标

## 5. 为什么玩家会用

1. 决策价值
- 判断“继续抽还是等下期”
- 判断“角色池还是武器池投入更划算”

2. 社交价值
- 分享卡片非常直观，能低成本展示账号手气与历史

3. 规划价值
- 垫池与保底状态可见，原石预算更容易做

## 6. 落地建议
优先做“数据分析 JSON 输出”，再做“卡片渲染”：
- 第一步保证指标口径稳定
- 第二步保证视觉展示（和截图类似）
